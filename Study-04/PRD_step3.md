# PRD Step 3: 사용자 프로필 및 레시피 저장

## 1. 개요

사용자 프로필을 생성하고, 추천받은 레시피를 저장·관리할 수 있는 기능을 추가하는 단계.
별도 서버 없이 브라우저의 `localStorage`를 활용하여 데이터를 영구 저장한다.

## 2. 목표

- 사용자 프로필 생성 및 관리 (이름, 알레르기, 선호도 등)
- 추천받은 레시피를 즐겨찾기로 저장
- 저장된 레시피를 다시 조회·삭제할 수 있다
- 프로필 정보를 레시피 생성 시 자동 반영한다

## 3. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | HTML, CSS, JavaScript (Vanilla) |
| 저장소 | `localStorage` (브라우저 로컬 스토리지) |
| 데이터 형식 | JSON |

## 4. 기능 요구사항

### 4.1 사용자 프로필

- 닉네임 입력
- 알레르기 / 못 먹는 재료 등록 (태그 형태)
- 식단 선호도 선택: 일반 / 채식 / 비건 / 저탄수화물 / 고단백
- 매운맛 선호: 안 매움 / 보통 / 매움 / 아주 매움
- 기본 인원수 설정
- 프로필 수정 및 삭제

### 4.2 레시피 저장

- 2단계 상세 레시피 화면에서 "저장" 버튼
- 저장 시 자동으로 날짜/시간 기록
- 메모 추가 가능 (예: "다음에 소금 줄이기")
- 중복 저장 방지 (같은 이름 레시피 경고)

### 4.3 저장된 레시피 관리

- 저장된 레시피 목록 보기 (카드 형태)
- 최신순 / 이름순 정렬
- 키워드 검색 (요리명, 재료)
- 개별 레시피 삭제
- 상세 레시피 다시 보기

### 4.4 프로필 정보 레시피 연동

- 레시피 생성 시 프로필의 알레르기 재료를 자동 제외
- 식단 선호도를 프롬프트에 자동 반영
- 매운맛 선호도 반영
- 기본 인원수 자동 선택

## 5. 데이터 구조

### 5.1 사용자 프로필

```json

  "nickname": "홍길동",
  "allergies": ["땅콩", "새우"],
  "dietType": "일반",
  "spiceLevel": "보통",
  "defaultServings": 2,
  "createdAt": "2026-04-07T12:00:00Z",
  "updatedAt": "2026-04-07T12:00:00Z"

```

```json
    "name": "배추전",
    "time": "20분",

    "description": "바삭한 배추전",
    "ingredients": {
      "have": ["배추", "계란", "양파"],
```

```
    ▼

```

## 9. 프로필 → 레시피 프롬프트 연동

프로필 정보가 있을 경우, 2단계 프롬프트에 자동 추가:

```
추가 조건:
- 알레르기 재료 제외: 땅콩, 새우
- 식단: 일반
- 매운맛 선호: 보통
```

## 10. 에러 처리

| 상황 | 처리 방법 |
|------|-----------|
| localStorage 용량 초과 | "저장 공간이 부족합니다. 오래된 레시피를 삭제해주세요" |
| 프로필 데이터 손상 | 기본값으로 초기화 후 재설정 안내 |
| 중복 레시피 저장 | "이미 저장된 레시피입니다. 덮어쓰시겠습니까?" 확인 |
| localStorage 미지원 | "이 브라우저에서는 저장 기능을 사용할 수 없습니다" 안내 |
