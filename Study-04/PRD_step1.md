# PRD Step 1: 냉장고 이미지 인식

## 1. 개요

냉장고 사진을 업로드하면 AI가 식재료를 자동으로 인식하는 웹 애플리케이션의 첫 번째 단계.
OpenRouter API의 `google/gemma-3-27b-it:free` 모델을 활용하여 이미지 속 재료를 분석한다.

## 2. 목표

- 사용자가 냉장고 사진을 업로드하면 AI가 식재료 목록을 추출한다
- 인식된 재료를 사용자가 확인하고 수정할 수 있다
- 2단계(레시피 생성)에 전달할 재료 데이터를 준비한다

## 3. 기술 스택

| 구분 | 기술 |
|------|------|
| Frontend | HTML, CSS, JavaScript (Vanilla) |
| API | OpenRouter API |
| AI 모델 | `google/gemma-3-27b-it:free` (이미지 인식) |
| 인증 | `.env` 파일 기반 API 키 관리 |

## 4. 기능 요구사항

### 4.1 이미지 업로드

- 파일 선택 버튼으로 이미지 업로드 (JPG, PNG, WEBP)
- 드래그 앤 드롭 지원
- 이미지 미리보기 표시
- 최대 파일 크기: 5MB

### 4.2 이미지 분석 (AI)

- 업로드된 이미지를 Base64로 인코딩
- OpenRouter API를 통해 `google/gemma-3-27b-it:free` 모델에 전송
- 프롬프트: 이미지 속 식재료를 JSON 배열 형태로 추출 요청
- 로딩 상태 표시 (스피너 또는 프로그레스)

### 4.3 재료 목록 표시 및 편집

- 인식된 재료를 태그/칩 형태로 표시
- 개별 재료 삭제 가능
- 재료 수동 추가 가능 (입력 필드)
- "다음 단계" 버튼으로 2단계 진행

## 5. API 호출 명세

### 요청

```
POST https://openrouter.ai/api/v1/chat/completions
```

```json
{
  "model": "google/gemma-3-27b-it:free",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "text",
          "text": "이 냉장고 사진에서 보이는 식재료를 모두 찾아서 JSON 배열로 반환해줘. 형식: [\"재료1\", \"재료2\", ...] JSON 배열만 반환하고 다른 텍스트는 포함하지 마."
        },
        {
          "type": "image_url",
          "image_url": {
            "url": "data:image/jpeg;base64,{BASE64_IMAGE}"
          }
        }
      ]
    }
  ]
}
```

### 응답 처리

- `choices[0].message.content`에서 JSON 배열 파싱
- 파싱 실패 시 정규식으로 재료명 추출 (폴백)
- 에러 시 사용자에게 재시도 안내

## 8. 보안 고려사항

- API 키는 `.env` 파일에서만 관리, Git에 포함하지 않음
- 프론트엔드에서 직접 API 호출 시 키 노출 위험이 있으므로, 프로토타입 단계에서는 허용하되 추후 백엔드 프록시 도입 검토
- 이미지 파일은 클라이언트에서만 처리 (서버 업로드 없음)

## 9. 에러 처리

| 상황 | 처리 방법 |
|------|-----------|
| 이미지 미지원 형식 | 업로드 전 MIME 타입 검증 |
| 파일 크기 초과 | 5MB 이상일 경우 경고 메시지 |
| API 429 (Rate Limit) | "잠시 후 다시 시도해주세요" 안내 + 재시도 버튼 |
| API 응답 파싱 실패 | 폴백: 텍스트에서 재료명 추출 시도 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" 안내 |
