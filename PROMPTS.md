## 🔵 1단계: 프로젝트 기초 구조 + 퀴즈 데이터

### 프롬프트

```
PRD를 기반으로 상식 퀴즈 게임의 1단계를 구현해 주세요.

## 이번 단계 목표
프로젝트 기초 구조 생성 및 전체 퀴즈 데이터 40문제 작성

## 구현 요청 사항

### 1. 파일 구조 생성
아래 구조로 파일을 생성해 주세요:
```
Study-03/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── app.js
│   ├── quiz.js
│   ├── ranking.js
│   └── data.js
```

### 2. index.html
- 시맨틱 HTML5 구조
- 모든 화면(메인, 퀴즈, 카테고리 전환, 결과, 순위)을 하나의 HTML 파일 안에
  섹션(div)으로 구분하고 display 전환 방식 사용
- 각 섹션 id: #home-screen, #quiz-screen, #category-transition-screen,
  #result-screen, #ranking-screen
- CSS/JS 파일 연결
- meta viewport 반응형 설정
- 한국어 lang 속성

### 3. data.js - 퀴즈 데이터 40문제
- 카테고리 4개: 한국사, 과학, 지리, 일반상식
- 각 카테고리 10문제씩 총 40문제
- 데이터 형식:
  ```js
  const quizData = {
    "한국사": [
      {
        question: "문제 텍스트",
        options: ["보기1", "보기2", "보기3", "보기4"],
        answer: 0  // 정답 인덱스
      },
      // ... 10문제
    ],
    "과학": [ ... ],
    "지리": [ ... ],
    "일반상식": [ ... ]
  };
  ```
- 난이도는 중학교~고등학교 수준의 상식
- 정답이 골고루 분포되도록 (answer 값이 0~3에 편중되지 않게)
- 문제와 정답이 사실에 부합하는지 반드시 확인

### 4. style.css - 기본 스타일
- CSS 변수로 색상 팔레트 정의 (--primary, --correct, --wrong, --bg, --text 등)
- 모바일 우선 반응형 (min-width: 768px 미디어쿼리)
- 화면 전환용 .hidden 클래스
- 기본 레이아웃: 가운데 정렬, 최대 너비 640px
- 깔끔하고 현대적인 카드 스타일 UI
- 버튼 호버/액티브 상태

### 5. app.js - 화면 전환 기본 뼈대
- 화면 전환 함수: showScreen(screenId)
- DOMContentLoaded 이벤트에서 초기화
- 각 화면 전환 이벤트 리스너 등록 (빈 핸들러도 OK, 2단계에서 구현)

## 이번 단계에서 제외
- 퀴즈 진행 로직 (2단계)
- 점수 계산 및 결과 표시 (2단계)
- 순위 저장/조회 (3단계)

## 확인 사항
- index.html을 브라우저에서 열면 메인 화면이 보여야 합니다.
- 닉네임 입력 필드와 "게임 시작", "순위 보기" 버튼이 표시되어야 합니다.
- 콘솔 에러가 없어야 합니다.
```

### ✅ 1단계 완료 기준
- [ ] 모든 파일이 생성됨
- [ ] index.html 브라우저에서 메인 화면 정상 표시
- [ ] 40문제 데이터가 data.js에 정확히 작성됨
- [ ] 콘솔 에러 없음

## 🟡 2단계: 퀴즈 진행 로직 + 결과 화면

### 프롬프트

```
1단계가 완료되었습니다. 이제 2단계를 구현해 주세요.

## 이번 단계 목표
퀴즈 진행 핵심 로직과 결과 화면 구현

## 구현 요청 사항

### 1. quiz.js - 퀴즈 진행 로직
게임의 핵심 로직을 구현해 주세요:

#### 게임 상태 관리
- 현재 카테고리 인덱스
- 카테고리 내 현재 문제 인덱스
- 카테고리별 점수 객체
- 카테고리 순서: ["한국사", "과학", "지리", "일반상식"]

#### 게임 시작
- 닉네임 입력 검증 (빈 값 방지, 공백만 있는 경우 방지)
- 게임 상태 초기화
- 첫 번째 카테고리의 첫 번째 문제 표시

#### 문제 표시
- 카테고리 이름, 현재 문제 번호(n/10) 표시
- 질문 텍스트 표시
- 4개 보기 버튼 동적 생성
- 전체 진행률 프로그레스바 (0~40 기준)
- 현재 점수 표시

#### 정답/오답 피드백
- 보기 클릭 시 즉시 판정
- 정답: 선택한 보기 초록색(--correct) + "✅ 정답입니다!" 메시지
- 오답: 선택한 보기 빨간색(--wrong) + 정답 보기 초록색 + "❌ 오답입니다!" 메시지
- 피드백 표시 후 모든 보기 버튼 비활성화 (중복 클릭 방지)
- "다음 문제 →" 버튼 표시

#### 다음 문제 이동
- 카테고리 내 다음 문제가 있으면 → 다음 문제 표시
- 카테고리의 마지막 문제(10번째)면 → 카테고리 전환 화면 표시
- 마지막 카테고리의 마지막 문제면 → 최종 결과 화면으로 이동

#### 카테고리 전환 화면
- 완료한 카테고리의 점수(n/10) 표시
- 다음 카테고리 이름과 아이콘 표시
- "계속하기 →" 버튼 클릭 시 다음 카테고리 시작
- 카테고리별 아이콘: 한국사 📜, 과학 🔬, 지리 🌍, 일반상식 💡

### 2. 최종 결과 화면
- 총점 표시: n / 40
- 정답률 퍼센트: (n/40 × 100)%
- 등급 계산 및 표시:
  - 36~40: S등급 🏆 "상식 마스터!"
  - 30~35: A등급 🥇 "상식 전문가!"
  - 24~29: B등급 🥈 "상식 우등생!"
  - 16~23: C등급 🥉 "조금만 더 노력!"
  - 0~15: D등급 📚 "공부가 필요해요!"
- 카테고리별 점수 막대 그래프 (CSS로 간단히)
- "다시 하기" 버튼 → 같은 닉네임으로 게임 재시작
- "메인으로" 버튼 → 메인 화면으로 이동

### 3. style.css 보완
- 보기 버튼 정답/오답 스타일 (.correct, .wrong 클래스)
- 프로그레스바 스타일
- 카테고리 전환 화면 스타일
- 결과 화면 등급 스타일 (등급별 색상)
- 카테고리별 점수 막대 그래프 스타일
- 피드백 메시지 애니메이션 (fadeIn)
- 부드러운 화면 전환 효과

## 이번 단계에서 제외
- 순위 저장/조회 (3단계)

## 확인 사항
- 게임 시작 → 40문제 전체 플레이 → 결과 화면까지 정상 동작
- 각 문제에서 정답/오답 피드백이 즉시 표시됨
- 카테고리 전환이 자연스럽게 동작
- 결과 화면에 총점, 등급, 카테고리별 점수가 정확히 표시됨
- 모바일/데스크톱 모두에서 레이아웃이 깨지지 않음
```

### ✅ 2단계 완료 기준
- [ ] 닉네임 입력 후 게임 시작 정상 동작
- [ ] 40문제 순차 진행 가능
- [ ] 정답/오답 즉시 피드백 동작
- [ ] 카테고리 전환 화면 정상 표시
- [ ] 최종 결과 화면에 총점, 등급, 카테고리별 점수 표시
- [ ] 다시 하기 / 메인으로 버튼 동작

## 🔴 3단계: 순위 시스템 + 최종 마무리

### 프롬프트

```
2단계가 완료되었습니다. 마지막 3단계를 구현해 주세요.

## 이번 단계 목표
순위(랭킹) 시스템 구현 및 전체 완성도 마무리

## 구현 요청 사항

### 1. ranking.js - 순위 시스템
LocalStorage를 활용한 순위 저장/조회 시스템:

#### 순위 저장
- 게임 완료 시 자동으로 순위 데이터 저장
- 저장 데이터: { nickname, totalScore, categoryScores, date }
- LocalStorage 키: "quizRanking"
- 배열 형태로 저장, 점수 내림차순 정렬
- 최대 10개까지만 유지 (11번째부터 삭제)
- 동점일 경우 최신 기록이 상위

#### 순위 조회/표시
- 순위 화면(#ranking-screen)에 테이블 형태로 표시
- 표시 항목: 순위, 닉네임, 총점, 날짜
- 기록이 없을 때 "아직 기록이 없습니다" 메시지 표시
- "메인으로" 버튼으로 돌아가기

#### 결과 화면 연동
- 결과 화면에서 순위 저장 완료 후 "현재 순위: n위" 표시
- 새 기록이 랭킹에 진입했으면 "🎉 순위권 진입!" 강조 표시

### 2. app.js 보완
- 순위 보기 버튼 이벤트 연결
- 게임 완료 → 순위 저장 흐름 연결
- 모든 화면 전환 최종 점검 및 누락된 이벤트 바인딩 보완

### 3. style.css 최종 마무리
- 순위 테이블 스타일 (1~3위 메달 아이콘 또는 색상 강조)
- 순위권 진입 강조 애니메이션
- 전체 디자인 통일성 점검
- 다크모드 및 라이트모드 자동 대응 (prefers-color-scheme)
- 모바일 터치 영역 최소 44px 확보
- 부드러운 전환 애니메이션 전체 적용
- 스크롤 필요 시 자연스러운 스크롤 처리

### 4. 최종 QA 체크리스트
아래 항목을 모두 확인하고, 문제가 있으면 수정해 주세요:
- 메인 → 게임 시작 → 40문제 → 결과 → 메인 전체 플로우
- 닉네임 빈 값 입력 시 적절한 안내
- 정답/오답 피드백의 정확성
- 카테고리 전환 시 점수 초기화되지 않는지
- 결과 화면 점수/등급 정확성
- 순위 저장 및 조회 동작
- 순위 10개 초과 시 잘림 처리
- "다시 하기"로 재시작 시 상태 완전 초기화
- 모바일/데스크톱 반응형 레이아웃
- 콘솔 에러 없음

## 확인 사항
- 전체 게임 플로우가 완벽히 동작
- 순위 저장 후 순위 화면에서 확인 가능
- 여러 번 플레이 시 순위가 올바르게 업데이트됨
- 모든 화면에서 모바일/데스크톱 정상 표시
- 브라우저 새로고침 후에도 순위 유지됨
```

### ✅ 3단계 완료 기준
- [ ] 게임 완료 후 순위 자동 저장
- [ ] 순위 화면에서 상위 10명 확인 가능
- [ ] 결과 화면에 현재 순위 표시
- [ ] 다크모드/라이트모드 자동 대응
- [ ] 전체 플로우 에러 없이 동작
- [ ] 모바일/데스크톱 반응형 정상
